# 아이디어 후보 정리 (7개)

> Low-code AI Startup Hackathon with Upstage — 팀 아이디어 선정용 비교표
> 작성일: 2026년 5월 27일 (수)

---

## 1. 대학생 학습 AX 솔루션 (팀원 A)

PPT·교과서·과제를 한꺼번에 올리면 자료 간 관계를 분석하고, 학생 상황(시험까지 남은 기간, 이해도, 목표)에 맞춰 학습 순서를 재배열. 문제 풀이 중 막힘을 감지하면 자동으로 보충학습으로 전환.

- **타겟:** PPT 중심 강의를 듣는 대학생(재무·회계·통계·공학 등) / 장기적으로 대학 LMS·교수학습지원센터(B2B)
- **API 활용:** Document Parse(자료 변환) + Solar LLM(개념 분석·문제 생성·막힘 감지·이해도 진단). Information Extract 활용은 명시적이지 않음
- **강점:** 시장이 크고 학생들의 실제 pain point(여러 자료를 오가는 비효율)에 정확히 닿음. "GPT 대비 차별점"이 명확히 정리됨
- **약점:** 스코프가 매우 큼(학습 경로 재배열 + 진척률 추적 + 막힘 감지 + 자료 크로스 분석…). 해커톤 5일 안에 안정적 데모 가능한지 의문. n8n 워크플로우가 비대해질 위험

---

## 2. AI 영상 제작 매니지먼트 툴 (팀원 A)

제품·타깃·플랫폼 정보를 입력하면 30~50개의 숏폼 광고 영상 기획표(훅·장면구성·이미지/영상 프롬프트·자막·검수기준)를 자동 생성. "영상 양산용 작업대".

- **타겟:** 광고대행사, 마케팅팀, 영상 제작사, 퍼포먼스 마케터(B2B SaaS)
- **API 활용:** Solar LLM 중심(기획안·프롬프트 생성). Document Parse / Information Extract와의 결합점이 약함
- **강점:** B2B 지불 의사 명확, 시장 트렌드(소재 대량 A/B 테스트)와 정확히 맞물림. 결과물 시각화가 데모에 좋음
- **약점:** **Upstage Document Parse / Information Extract를 핵심으로 쓰지 않음** → 심사 20점짜리 "Upstage API 활용도"에서 불리. "OpenAI API 또는 LLM API"라고 본인이 명시할 정도로 Upstage 의존성이 약함

## 3. 기업향 CS 챗봇 할루시네이션 가드레일 (팀원 B)

챗봇이 생성한 답변 초안을 사내 정책 PDF와 실시간 교차 검증. 금액·기간 같은 핵심 조건이 약관과 다르면 자동 차단 + 관리자 대시보드에 위험 알림.

- **타겟:** 금융·보험·이커머스(보안/정확성이 중요한 B2B)
- **API 활용:** Document Parse(약관 구조화) + Solar LLM(RAG 답변 생성) + Information Extract(금액·기간 추출 후 대조) — 세 API 모두 핵심
- **강점:** API 활용 구조가 깔끔하고 논리적. "AI 안전성"이라는 명확한 가치 제안. B2B 수익 모델 설명도 구체적
- **약점:** 데모의 시각적 임팩트가 약할 수 있음(텍스트 대조의 정확성을 5분 발표에서 보여주기 어려움). 타겟 산업(금융·이커머스)이 무겁고 대학생 해커톤에서 검증된 인사이트인지 의문

---

## 4. 다중 화면 기반 실시간 AI 협업 문서 작성 도구 (팀원 B)

왼쪽엔 여러 참고 문헌(PDF), 오른쪽엔 작성 화면. 사용자가 글 쓰다가 막히면 AI가 왼쪽 자료를 분석해 실시간으로 이어 써주고, 결과물 문장을 클릭하면 원본 문서의 정확한 위치로 자동 이동 + 형광펜.

- **타겟:** 이공계 연구원·대학생, 시장 데이터를 다루는 실무자
- **API 활용:** Document Parse(문서의 위치 좌표값 추출) + Solar LLM(문맥 기반 텍스트 생성). 위치 좌표 추적이 핵심
- **강점:** "출처 검증의 시각적 한계"라는 진짜 문제를 잡음. 데모 임팩트 큼(클릭 한 번에 형광펜 점프). 차별성이 매우 뚜렷
- **약점:** **n8n + Lovable로 구현 가능한 범위를 벗어남.** "다중 화면 동기화", "위치 좌표 기반 형광펜", "실시간 양방향 통신" 등은 사실상 풀스택 웹앱이 필요. 해커톤 도구 스택과 미스매치. Low-code의 한계를 정통으로 부딪힘

---

## 5. 처방전/약봉투 복약 가이드 AI (현욱 A)

약봉투/처방전 사진 → 약품명·용법·용량 자동 추출 → 쉬운 말 복약 설명 + 약물 상호작용 + 음식 궁합. 고령 부모용 "가족 복약 비서".

- **타겟:** 고령 부모를 둔 30~40대(B2C) / 약국·요양시설(B2B)
- **API 활용:** Document Parse + Information Extract + Solar LLM 셋 다 핵심
- **강점:** 초고령사회 시의성, API 파이프라인이 자연스럽고 안정적, 데모 직관적
- **약점:** "문서 올리면 분석" 패턴이 다른 팀과 겹칠 가능성, 의료 정확성 검증 이슈

## 6. 피싱/사기 문서 검증기 (현욱 B)

의심 문서(가짜 공문서, 사칭 안내문)를 올리면 발신기관·사건번호·연락처·요구행동을 추출하고, 위조 신호를 분석해 신호등으로 위험도 표시.

- **타겟:** 전국민(특히 고령자 가족, B2C) / 금융권(B2B)
- **API 활용:** Document Parse + Information Extract + Solar LLM 셋 다 핵심. 특히 "사기 탐지"라는 반대 방향 맥락이 독특
- **강점:** 2025년 피해 1.2조 원이라는 압도적 문제 크기, 사회적 임팩트, 데모 임팩트 강함, 참신성
- **약점:** 위조 패턴 분석의 정확성을 데모 5분 안에 설득력 있게 보여줄 수 있는지

## 7. 자취방 안전 검증 — 등기부등본 × 임대차계약서 크로스체크 (현욱 D)

등기부등본 + 임대차계약서 두 문서를 동시에 올리면 소유자 일치·근저당 대비 보증금 안전성·불리한 특약을 크로스 분석해 전세사기 위험도 표시.

- **타겟:** 첫 자취 대학생·사회초년생(B2C) / 부동산 플랫폼(B2B)
- **API 활용:** Document Parse(×2) + Information Extract(×2, 서로 다른 스키마) + Solar LLM(크로스 분석) — 7개 중 API 활용 깊이 최고
- **강점:** 두 문서 크로스라는 기술적 차별성, 전세사기 공감대
- **약점:** 구현 난이도 최고. 해커톤 기간 내 안정적 데모 가능 여부가 관건

Upstage API 활용도(20점)가 심사에서 가장 큰 단일 항목인데, 이 기준으로 보면 2번(영상 제작)과 1번(학습 AX)은 상대적으로 불리하고, 3·5·6·7번은 세 API가 자연스럽게 결합되는 구조. 4번은 컨셉은 좋지만 n8n+Lovable 스택과 미스매치가 크다는 게 가장 걸리는 지점.
