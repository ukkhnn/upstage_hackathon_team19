### 1-3. 필수 기술 스택

**Upstage API (AI 두뇌 — 무상 크레딧 제공)**
- **Solar LLM:** 자연어 처리, 텍스트 생성, 분석, 추론. Solar Pro 3가 최신 모델(MoE 102B, 활성 12B, 128K 컨텍스트). 한국어 성능이 특히 강점.
- **Document Parse:** PDF, 스캔 이미지(tiff, jpeg 등), MS Office 문서를 HTML 또는 Markdown으로 변환. 동종 제품 중 최고 정확도(TEDS 94.48). 내부적으로 파일 변환기 + OCR + 레이아웃 분석 파이프라인으로 구성.
- **Document Classify:** 문서를 사전 정의 카테고리로 분류. 모델 학습 불필요, 100개 이상 클래스 지원.
- **Information Extract:** 문서에서 원하는 정보를 구조화 추출. 기존 OCR과 달리 맥락과 의도를 이해하며, 스키마에 구애받지 않는 유연한 적응. LLM 기반 에이전틱 파이프라인(문서 라우터 → 파서 → 추출기 → 검증기).

**n8n (자동화 백엔드)**
- 코딩 없이 드래그앤드롭으로 워크플로우 구성
- Upstage API와 각종 서비스(Google Sheets, Notion, Slack 등)를 시각적으로 연결
- JSON 형태의 워크플로우 설계 코드를 최종 제출해야 함

**Lovable (프론트엔드 UI 빌더)**
- 텍스트 입력만으로 고품질 웹/앱 UI 생성
- 디자이너/기획자도 사용 가능
- 유료 플랜 크레딧이 참가자에게 제공됨
- 퍼블리싱 주소를 최종 제출해야 함

## 3. 현욱이 제출한 아이디어 요약

### 아이디어 A: 처방전/약봉투 복약 가이드 AI
약봉투나 처방전 사진을 올리면 약품명·용법·용량을 자동 추출하고, 쉬운 말로 복약 설명 + 약물 간 상호작용 주의사항 + 음식 궁합까지 알려주는 서비스. 고령 부모님 약을 대신 확인해주는 "가족용 복약 비서".

- **API 활용:** Document Parse + Information Extract + Solar LLM (세 개 모두 핵심)
- **타겟:** 고령 부모를 둔 30~40대 / 약국·요양시설(B2B)
- **강점:** 초고령사회 시의성, API 활용도 높음
- **약점:** 의료 정확성 검증 이슈, "문서 분석" 패턴이 다른 팀과 겹칠 수 있음

### 아이디어 B: 피싱/사기 문서 검증기 — "이거 진짜야?"
카톡으로 받은 의심 문서를 업로드하면 발신기관·사건번호·연락처 등을 자동 추출하고, AI가 위조 신호를 분석해 위험도(안전/주의/위험)와 의심 포인트를 알려주는 서비스.

- **API 활용:** Document Parse + Information Extract + Solar LLM (세 개 모두 핵심, 특히 "사기 탐지"라는 독특한 맥락)
- **타겟:** 전국민(특히 고령자 가족) / 금융권(B2B)
- **강점:** 2025년 피해 1.2조 원이라는 압도적 문제 크기, "문서 분석으로 사기 탐지"라는 반대 방향의 참신성, 사회적 임팩트, 데모 임팩트
- **약점:** 위조 패턴 분석의 정확성을 데모에서 충분히 보여줄 수 있는지

### 아이디어 D: 자취방 안전 검증 — 등기부등본 × 임대차계약서 크로스체크
두 문서를 동시에 올리면 소유자 일치 여부, 근저당 대비 보증금 안전성, 불리한 특약 조항을 자동 크로스 분석해서 전세사기 위험도를 보여주는 서비스.

- **API 활용:** Document Parse(×2) + Information Extract(×2, 서로 다른 스키마) + Solar LLM(크로스 분석) — 4개 아이디어 중 API 활용 깊이 최고
- **타겟:** 첫 자취 대학생·사회초년생 / 부동산 플랫폼(B2B)
- **강점:** 두 문서 크로스 분석이라는 기술적 차별성, 전세사기 예방이라는 공감대
- **약점:** 구현 난이도가 가장 높음 (해커톤 1~2일 내에 안정적 데모 가능 여부)

## 4. 참고 사항

### 4-1. 현욱 개인 배경
- 서강대 4학년 컴퓨터공학과, 졸업 2027년 2월 예정
- ML 이론 기반(PyTorch, Transformer from scratch), CS 기초(OS/Pintos, C/C++, Python) 탄탄
- 실무 LLM 애플리케이션 경험은 제한적 — 이번 해커톤이 실전 경험 축적 기회
- n8n + Upstage API + Lovable 조합은 이번이 처음
- Mac Mini M4 Pro(64GB) 메인 개발 머신

### 4-2. 전략적 고려사항
- 개발자 최소 2명 + 3인 팀이므로 역할 분담에 어느 정도 여유. 다만 4인 → 3인으로 줄어든 만큼 일정은 타이트하게 관리 필요
- 심사에서 기술(50점)이 가장 크므로, Upstage API 활용도 + 구동 완성도가 최우선
- 발표 현장에서 반드시 오류 없이 동작해야 함 — 과도하게 복잡한 파이프라인보다는 안정적으로 돌아가는 구조가 유리
- 상호평가(10점)가 있으므로 데모 시연의 시각적 임팩트도 중요
- 5분 발표 + 3분 QnA 구조이므로 발표 자료는 핵심만 압축

### 4-3. 아직 확인 필요한 사항
- 우리 팀 번호 및 팀원 상세 정보
- 비즈니스 기획서 템플릿 양식 (사전 제공된다고 했는데 아직 미확인)
- Upstage API 크레딧 지급 방식 및 한도
- Lovable 크레딧 지급 방식
- 팀원 중도 하차에 따른 운영진 공지사항 여부
- n8n 사전 학습 강의 링크 및 이수 여부
